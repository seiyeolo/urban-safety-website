# 벤치마킹 브리프 — urban-safety-website

## 목적

`urban-safety-website`의 UI/UX를 기관형 웹사이트 수준으로 끌어올리기 위한 벤치마킹을 수행한다.

이번 단계의 목표는 "예쁜 사이트 수집"이 아니라, 현재 코드베이스에 실제로 반영 가능한 패턴을 선별하는 것이다.

## 프로젝트 성격

- 기관명: 대전경실련 도시안전디자인센터
- 성격: 경실련 산하 시민공익단체
- 핵심 기능: 센터 소개, 교육 안내, 민간자격 소개, 공지/자료실, 문의 접수, 간단한 관리자 CRUD
- 우선 가치: 신뢰감, 공공성, 가독성, 기관다운 무게감
- 비선호: 스타트업 랜딩페이지, 과한 인터랙션, 실험적 아트디렉션, 앱 스타일 대시보드 미학

## 벤치마킹 소스 우선순위

1. Awwwards
2. Dribbble
3. 필요시 Mobbin 또는 실제 기관 사이트

Unsplash는 UI/UX 벤치마킹 소스가 아니라 이미지 소스이므로 이번 단계의 주력 참고처로 사용하지 않는다.

## 수집 대상

아래 유형의 사례를 우선 수집한다.

- Corporate / Business / Editorial 계열의 신뢰형 웹사이트
- 공익단체, 재단, 연구소, 교육기관, 협회, 정책기관에 가까운 톤의 사이트
- 정보량이 많지만 위계가 명확한 사이트
- 모바일에서도 텍스트 가독성이 무너지지 않는 사이트

## 제외 대상

아래 사례는 원칙적으로 제외한다.

- WebGL, 3D, 과한 모션이 중심인 사이트
- 이미지 포트폴리오 중심 사이트
- SaaS 대시보드형 랜딩페이지
- 지나치게 젊고 트렌디한 D2C 브랜드 사이트
- 한국어 정보형 기관 사이트에 적용하기 어려운 컨셉샷

## 반드시 봐야 할 UI 영역

각 사례를 볼 때 아래 항목별로 나눠서 분석한다.

1. 헤더
2. 히어로 섹션
3. 신뢰도 지표/소개 블록
4. 카드형 콘텐츠 구획
5. 공지/리스트형 레이아웃
6. CTA 버튼 스타일
7. 페이지 제목부와 본문 폭
8. 모바일 메뉴와 모바일 여백
9. 푸터 정보 구조

## 분석 방식

각 참고 사례마다 아래를 기록한다.

- 무엇이 좋은가
- 왜 우리 사이트에 맞는가
- 그대로 쓰면 안 되는 점은 무엇인가
- 어느 페이지/컴포넌트에 적용할 수 있는가

즉, "멋있다"가 아니라 "어디에 어떻게 옮길 수 있는가" 기준으로 정리한다.

## 산출물 형식

최종 결과는 아래 3개 섹션으로 정리한다.

### 1. 레퍼런스 목록

- 총 8~12개
- 각 항목당: 이름, 출처, 링크, 한 줄 요약

### 2. 패턴 추출표

아래 형식의 표로 정리한다.

| 영역 | 참고 사례 | 관찰 포인트 | 우리 사이트 적용안 |
|------|-----------|-------------|---------------------|

### 3. 최종 디자인 방향

다음 내용을 포함한다.

- 우리 사이트에 맞는 톤앤매너 3문장
- 버려야 할 현재 디자인 요소
- 유지해도 되는 요소
- 홈 화면 개편 우선순위
- 공통 컴포넌트 개편 우선순위

## 실제 반영을 고려한 제한

- 현재 프로젝트는 Next.js + Tailwind 기반이다
- 즉시 전체 리빌드보다 점진적 개편이 유리하다
- 먼저 홈 화면과 공통 UI부터 개선하는 흐름을 전제로 한다
- 복잡한 모션보다 타이포, 색, 레이아웃, 간격, 카드 구조 개선이 우선이다

## 기대 결과

벤치마킹 결과를 읽으면 바로 아래 결론이 나와야 한다.

- 메인 페이지를 어떤 무드로 바꿀지
- 헤더/푸터/버튼/카드를 어떤 계열로 통일할지
- 어느 레퍼런스의 어느 부분을 차용할지
- 무엇은 절대 따라하지 말아야 할지
