## 1. 결정 요약

기존 도시안전 웹사이트는 최초 기획안처럼 **심플한 기관/교육 안내형 홈페이지**로 유지한다.

온라인 강의 기능은 기존 사이트 안에 계속 덧붙이지 않고, **별도의 온라인강의 사이트/LMS로 분리 구축**한다.

기존 웹사이트에는 온라인강의 사이트로 이동하는 **명확한 배너/CTA**만 둔다.

> 결론: `기관 홈페이지`와 `온라인 강의실`을 이원화한다.

---

## 2. 결정 이유

### 2-1. 기존 사이트의 목적을 흐리지 않기 위해

현재 기존 사이트의 역할은 다음이다.

- 센터 소개
- 민간자격 안내
- 교육 과정 안내
- 공지/자료실
- 문의/신청 안내
- 기관 신뢰 형성

여기에 LMS 기능을 모두 넣으면 사이트 목적이 흐려진다.

### 2-2. 수강생 UX를 단순하게 만들기 위해

온라인 수강생은 아래를 빠르게 찾아야 한다.

```text
로그인 → 내 강의실 → 강의 재생 → 자료 다운로드 → 진도 확인
```

기관 홈페이지 안에 강의실이 섞이면 메뉴가 복잡해지고, 특히 중장년 수강생이 강의실을 찾기 어려워질 수 있다.

### 2-3. 운영/개발 복잡도를 줄이기 위해

기관 홈페이지와 LMS는 필요한 기능이 다르다.

| 구분 | 기존 웹사이트 | 온라인 강의 사이트 |
|---|---|---|
| 핵심 목적 | 안내/신뢰/문의 | 수강/자료/진도/관리 |
| 사용자 | 일반 방문자, 기관 담당자 | 수강생, 강사, 관리자 |
| 기능 | 콘텐츠 페이지, 공지, 문의 | 로그인, 수강권한, 영상, 자료, 진도 |
| 위험 | 문구/법적 고지 | 개인정보, 영상권한, 결제, 수료 기록 |
| 디자인 | 심플한 기관형 | 앱/대시보드형 LMS |

따라서 한 곳에 모두 넣기보다 분리하는 것이 안전하다.

## 3. 최종 구조

```text
[기존 도시안전 웹사이트]
- 목적: 센터 소개, 자격 안내, 교육 안내, 공지, 문의
- 유지 방향: 심플하고 신뢰감 있는 홈페이지
- 역할: 온라인 강의실로 보내는 입구

        ↓ 배너 / CTA

[신규 온라인강의 사이트]
- 목적: 수강생 학습
- 기능: 회원가입, 로그인, 내 강의실, 영상, 자료, 진도, 관리자 LMS
- 디자인: 대시보드형, 수강생 중심, 앱처럼 단순한 UI
```

---

## 4. 기존 웹사이트에 남길 것

기존 사이트는 아래 정도만 유지한다.

1. 센터 소개
2. 자격증 안내
3. 교육 과정 소개
4. 공지사항
5. 자료실/기본 안내
6. 문의/교육 신청
7. 온라인 강의실 이동 배너

### 온라인 강의 배너 예시

```text
온라인으로 수강 중이신가요?
도시안전 온라인 강의실에서 이어서 학습하세요.
[온라인 강의실 바로가기]
```

또는:

```text
오프라인 교육 수강생 전용
강의자료·동영상·진도 확인은 온라인 강의실에서 가능합니다.
[내 강의실 입장]
```

## 5. 신규 온라인강의 사이트에 만들 것

신규 LMS 사이트는 아래 구조로 설계한다.

```text
/
온라인 강의실 홈

/login
수강생 로그인

/signup
회원가입 또는 초대 등록

/dashboard
내 강의실

/courses
내가 들을 수 있는 과정

/courses/[slug]
과정 상세

/learn/[courseSlug]/[lessonSlug]
강의 플레이어

/materials
강의자료

/admin
관리자 대시보드

/admin/courses
강의 관리

/admin/lessons
차시/영상/자료 관리

/admin/enrollments
수강생 배정

/admin/progress
진도/수료 관리
```

## 6. 구현 방향

### Phase 1 — 기존 사이트 정리

- 기존 사이트는 기획안 대로 심플한 구조로 정리
- 오늘 추가한 mock LMS 페이지는 운영 노출에서 제외 또는 별도 실험 링크로 보류
- 온라인 강의실 배너/CTA 추가
- 사이트 목적을 “기관/교육 안내”로 명확화

### Phase 2 — 신규 LMS 설계

- 신규 사이트명/도메인 후보 정리
- IA/화면흐름/DB schema 설계
- 회원가입/로그인/수강권한/자료/진도/관리자 흐름 확정

### Phase 3 — 신규 LMS 구현

- Supabase Auth/DB 연결
- 수강생 계정/강의 배정
- 영상/자료 업로드 구조
- 강의 플레이어
- 진도 저장
- 관리자 LMS
- 테스트/보안/배포

## 7. 민호/준호 작업 지시 방향

### 민호에게

```text
Urban Safety 사이트 이원화 전략으로 전환한다.
기존 웹사이트는 심플한 기관 안내형으로 유지하고, 온라인 강의 기능은 별도 LMS 사이트로 분리 설계한다.
오늘 만든 mock LMS는 방향 참고자료로만 사용하고, 기존 사이트 안에 복잡하게 병합하지 않는다.
```

### 준호/Claude Code/Codex에게

```text
기존 urban-safety-website는 심플 홈페이지로 리팩토링한다.
LMS 기능은 별도 앱/브랜치/프로젝트 후보로 분리한다.
기존 사이트에는 온라인 강의실 이동 배너와 CTA만 추가한다.
실제 회원가입, DB, 영상, 자료, 진도, 관리자 LMS는 신규 LMS 프로젝트에서 설계/구현한다.
```

## 8. 소라 안전 게이트

분리 전략에서도 아래는 별도 승인 전 금지한다.

- 실제 수강생 개인정보 업로드/저장
- 실제 영상 업로드/외부 호스팅 연결
- 결제 활성화
- Supabase 운영 DB schema 변경
- production 강의 서비스 오픈
- 외부 수강생 초대 발송
- 자격/수료/합격률 단정 문구
- 개인정보처리방침/환불/약관 확정 문구 변경

---

## 9. 대표 보고용 한 줄

> 기존 웹사이트는 심플한 기관 안내형으로 유지하고, 온라인 강의실은 별도 LMS 사이트로 분리해 수강생이 바로 로그인하고 강의를 찾을 수 있게 만드는 전략으로 확정한다.
