## 1. 한 줄 결론

현재 사이트는 **교육 홍보·신청형 웹사이트**에 가깝고, 앞으로는 패스트캠퍼스식 **강의 탐색·상세·결제/신청·내 강의실·진도관리** 구조와 Hugging Face/GitHub식 **학습 로드맵·실습형 커리큘럼**을 결합한 **소규모 LMS형 교육 포털**로 재구성하는 것이 맞다.

## 3. 벤치마크 요약

### 3-1. FastCampus

관찰 포인트:
- 상단 검색창과 카테고리 탐색이 강하다.
- 강의 카드에 **대표 이미지, 카테고리, 제목, 수강기간/시간, 난이도, 가격, 온라인 여부**가 들어간다.
- 이벤트/할인/마감/혜택 CTA가 강하게 노출된다.
- 로그인, 기업교육, 커뮤니티 등 부가 메뉴가 분리되어 있다.

우리에게 적용할 점:
- 도시안전 강의도 카드형으로 `과정명 / 대상 / 총 시간 / 수강 방식 / 수료 기준 / 상태`를 한눈에 보여준다.
- 자격증 과정은 상세 페이지에 **커리큘럼, 강사, 수강 대상, 수료/검정 기준, 비용, 환불, FAQ**를 고정 구조로 둔다.
- 단, 공익/안전교육 사이트이므로 과도한 세일즈/할인 카피는 줄이고 신뢰·공공성 중심으로 조정한다.

### 3-2. Hugging Face Learn

관찰 포인트:
- `/learn`에서 여러 코스를 카드형으로 나열한다.
- 각 코스는 명확한 주제와 설명이 있고, 개발자/학습자에게 바로 “어떤 코스인지”가 보인다.
- Docs와 Course가 연결되어 있어 학습자료를 지식문서처럼 읽을 수 있다.

우리에게 적용할 점:
- “온라인 교육”을 단순 신청 페이지가 아니라 **학습 카탈로그**로 바꾼다.
- 강의별로 `영상 + 요약자료 + 체크리스트 + 실습/퀴즈`를 묶는다.
- 안전교육 특성상 강의 후 현장 적용 체크리스트를 제공한다.

### 3-3. GitHub Skills

관찰 포인트:
- GitHub Skills의 개별 코스는 “누가 듣는지 / 무엇을 배우는지 / 무엇을 만드는지 / 1시간 이내 시작”처럼 실습 단위가 명확하다.
- README 기반으로 단계형 실습을 제공한다.
- 수강생이 단순 시청자가 아니라 직접 따라 하도록 설계되어 있다.

우리에게 적용할 점:
- 동영상만 올리지 말고 각 차시마다 **따라 하기 과제**를 붙인다.
- 예: 보이스피싱 예방지도사 과정에서는 “의심 문자 판별하기”, “고령층 대상 5분 설명안 작성하기”, “피해 발생 시 대응 플로우 정리하기” 같은 실습을 붙인다.
- 수료 조건은 `진도율 + 퀴즈 + 과제 제출`로 설계한다.

### 3-4. edX

관찰 포인트:
- 과목/목표/레벨/기간 기준으로 코스를 탐색한다.
- 대학/기관 기반 신뢰를 강조한다.
- self-paced, 2~6주 같은 기간 정보가 명확하다.

우리에게 적용할 점:
- 도시안전디자인센터는 신뢰가 중요하므로 **기관성, 자격 정보, 수료 기준, 교육 기간**을 명확히 노출한다.
- 검색/필터는 처음부터 크게 만들기보다 `자격증 과정 / 시민안전 특강 / 기관교육 / 보수교육` 정도의 4분류부터 시작한다.

## 10. 오프라인 수강생 온라인 전환에서 특히 중요한 것

1. **초대 방식**  
   수강생에게 “회원가입하세요”가 아니라, 관리자 등록 후 초대 링크 또는 임시 비밀번호 방식이 편하다.

2. **모바일 우선**  
   오프라인 수강생은 PC보다 휴대폰으로 볼 가능성이 높다. 강의 플레이어와 자료 다운로드가 모바일에서 좋아야 한다.

3. **기수/반 관리**  
   같은 강의라도 1기, 2기, 오프라인 보충반, 기관 단체반으로 나뉠 수 있다.

4. **수료 기준 명확화**  
   진도율 80%, 퀴즈 60점, 과제 제출 등 기준을 페이지에 명확히 써야 한다.

5. **민간자격/환불/개인정보 고지**  
   교육 플랫폼화하면 개인정보와 결제/환불/자격 고지가 더 중요해진다.
